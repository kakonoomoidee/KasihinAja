# v1.3.0 - Furious Kettle

> Released: 2026-03-29

Critical infrastructure corrections across the full stack. Live donations now save correctly, OBS alerts display media and voice notes reliably, and milestone progress accumulates without surprise resets.

## Bug Fixes

### Backend: EVM Listener Overhaul

- **Fixed stateless token decoding**: `evmListener.js` was calling `DonationIntent.findByPk()` against a database table that the intent system never writes to. Intents are stateless HMAC-signed tokens. Replaced the DB lookup with `verifyToken()` from `utils/token.js`. This was the root cause of all media data arriving as `NULL` in the database and `null` in WebSocket payloads.
- **Fixed Wei to ETH validation**: On-chain `amount` arrives from the Smart Contract in Wei (`bigint`). The listener now converts it using `ethers.formatEther(amount)` before any comparison or arithmetic, eliminating false validation mismatches.
- **Fixed milestone math**: Replaced plain addition with explicit `parseFloat()` casts on both operands so floating-point milestone values accumulate correctly: `parseFloat(profile.milestone_current || 0) + parseFloat(ethAmount)`.
- **Removed milestone auto-reset**: The block that zeroed `milestone_current` when the goal was reached has been removed. Progress now accumulates indefinitely until the streamer explicitly resets via the dashboard. The visual bar is capped at 100% using `Math.min(100, ...)` on the frontend.
- **Fixed `MILESTONE_UPDATE` broadcast value**: The broadcast was reading `profile.milestone_current` after it had potentially been zeroed by the auto-reset block. The broadcast now forwards the calculated `newCurrent` value directly.

### Backend: Sequelize Model and Insertion Mapping

- **Confirmed `DonationHistory` model**: Verified that `media_url` and `vn_url` are explicitly defined as `DataTypes.TEXT` in `DonationHistory.js`. Sequelize silently drops undefined columns during `.create()`, so explicit definitions are required.
- **Fixed insertion mapping**: `evmListener.js` now correctly maps `mediaPayload.youtube_url` to `media_url` and `mediaPayload.vn_data || mediaPayload.vn_url` to `vn_url` during `DonationHistory.create()`.
- **Fixed `vn_data` storage**: Voice notes are recorded as Base64 data URIs from the donor browser. The listener now prioritises `vn_data` over `vn_url` when writing to the `vn_url` database column.

### Backend: WebSocket Payload Standardisation

- **`profileController.js` (test alert)**: Changed broadcast type from `NEW_DONATION` to `VERIFIED_DONATION` to match what `AlertOverlay.jsx` listens for. Added `youtube_url: null` and `vn_data: null` to the test payload.
- **`historyController.js` (replay)**: `getHistory` now maps DB column names to frontend-expected aliases: `youtube_url: record.media_url`, `vn_data: record.vn_url`. The replay broadcast includes `youtube_url` and `vn_data` at the root level.
- **`MILESTONE_UPDATE` payload**: Both `evmListener.js` and `profileController.js` now broadcast `milestone_current` and `milestone_target` numeric values inside the payload instead of an empty object.

### Frontend: OBS Alert Overlay

- **Bulletproof media key extraction**: `AlertCard` resolves `youtube_url` from `alert.media_data?.youtube_url || alert.youtube_url || alert.media_url` and `vnSrc` from all possible key variants to handle both the flattened broadcast shape and the nested intent shape.
- **Voice Note waveform visualizer**: Added `VoiceNoteVisualizer` component rendering 12 animated bars with a `vnBar` CSS keyframe animation. Each bar has a pre-seeded delay and duration for independent movement. Rendered inside both Classic and Minimalist alert templates when a voice note is active.
- **Dynamic alert dismissal**: Replaced the hardcoded 15-second VN timeout with an `onEnded` event listener on the `<audio>` element. Voice note alerts now dismiss immediately when the audio finishes. YouTube-only alerts retain the 15-second window. Text-only alerts dismiss after 5 seconds.
- **`hasMedia` display timer fix**: The timer branch now checks all key paths including `next.media_url` and `next.vn_data` so the correct display duration is selected regardless of payload shape.

### Frontend: Milestone Overlay

- **`MilestoneOverlay.jsx`**: WebSocket handler for `MILESTONE_UPDATE` now reads `data.payload.milestone_current` and `data.payload.milestone_target` to update state directly without an HTTP round-trip.
- **Progress bar cap**: All milestone progress bars across `MilestoneOverlay.jsx`, `Dashboard.jsx`, and `PublicProfile.jsx` use `Math.min(100, (current / target) * 100)` to prevent visual overflow.
